# APR Day 8 Founder Walkthrough

1. What Day 8 proves: the technical APR lesson shell can be opened, navigated, recorded, optionally transcribed, optionally given technical model audio and technical feedback, retried, reflected on, and ended with a clear session-only summary.
2. What Day 8 does not prove: final Lesson 1 content, learning progress, evidence, scoring, correctness, proficiency, improvement, Professor Gabriel feedback, or public readiness.
3. Open the private or local staging environment supplied by the operator. If no private link has been provided, the hosted staging step is not ready yet.
4. Log in with an authorized test account.
5. Open `/apr/primeira-conexao/lessons/enter-the-connection`.
6. Move through the five lesson steps by pressing Continue: Orientation, Information, Single-choice interaction, Microphone capture, Reflection.
7. Allow microphone access only when the browser asks after you press Start recording.
8. Record the Original attempt by pressing Start recording and speaking a short test phrase.
9. Play it back using the Original attempt audio controls.
10. Record an optional Latest retry by pressing Record another attempt.
11. Request an Original transcript by pressing Generate transcript draft.
12. Edit the draft if needed and press Confirm reviewed transcript.
13. Generate technical model audio by pressing Generate technical model audio.
14. Request technical feedback by pressing Generate technical feedback after confirming the Original transcript.
15. Record a post-feedback retry by pressing Record another attempt after feedback appears.
16. Enter any optional note in the final Reflection box.
17. Reach the session-closure screen by pressing Continue from Reflection.
18. Interpret the summary rows as technical activity only: Captured means audio exists in this browser session; Confirmed means the transcript was confirmed in this session; Generated means model audio loaded; Ready means technical feedback loaded; Post-feedback retry describes whether a retry was captured after feedback.
19. Use Back to reflection to return to the note without losing session state.
20. Test cancelled Restart by pressing Restart technical session and cancelling the browser confirmation; the closure screen should remain.
21. Test confirmed Restart by pressing Restart technical session and accepting the browser confirmation; the lesson should return to Orientation and clear the session.
22. Exit by pressing Exit to APR module; the browser should navigate to `/apr/primeira-conexao`.
23. Collect screenshots or notes for: recording step, transcript confirmation, model audio, feedback, closure summary, cancelled Restart, confirmed Restart, and exit.
24. Count as bugs: closure appears before final Continue, summary claims learning, Original recording disappears without Restart, optional tools are required, or exit sends a completion claim.
25. Intentional limitations: placeholder content, generated technical audio, fixed technical feedback, no score, no evidence, no progress, no permanent records, and no hosted link unless deployed by an operator.
26. Report results with the browser, device, route, exact step, expected behavior, actual behavior, and screenshot when possible.
27. Founder-test readiness: READY WITH CONSTRAINTS.
