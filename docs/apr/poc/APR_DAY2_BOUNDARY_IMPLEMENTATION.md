Create a small, isolated technical boundary for Academia Português Reconectado (APR) inside FreeLingo without building a lesson, changing FreeLingo learning behavior, or using AI, speech, billing, placement, study-plan, XP, or streak systems.

## 5. Backend route

- Route: `GET /api/apr/modules/primeira-conexao`
- Package: `backend/app/apr/`
- Response: fixed typed metadata for the APR Day 2 technical boundary.
- The response is not generated by AI and does not read lesson, placement, study-plan, progress, billing, conversation, STT, or TTS state.

## 6. Authentication behavior

The backend route uses FreeLingo's existing authenticated-user dependency, `get_current_user`. This proves APR can reuse account login without reusing CEFR placement or study-plan generation.

Unauthenticated backend requests receive the existing authentication failure behavior from FreeLingo's auth dependency.

## 7. Frontend route and auth-only layout

- Browser route: `/apr/primeira-conexao`
- Page file: `frontend/src/app/(apr)/apr/primeira-conexao/page.tsx`
- Layout file: `frontend/src/app/(apr)/layout.tsx`
- APR now has a separate auth-only frontend layout outside the inherited `(app)` route group.
- The APR layout reuses only the refresh-token endpoint, auth store, access-token behavior, `/api/auth/me`, user mapping utilities, login redirect, and existing loading component.
- APR does not require FreeLingo onboarding and does not inspect `learning_goals`.
- APR does not inherit the FreeLingo sidebar, language switcher, subscription state, feedback count loading, or learning navigation.
- The page calls `GET /api/apr/modules/primeira-conexao` through the existing `apiFetch` helper.
- The page displays loading, success, disabled-environment, and technical error states.

## 8. Feature flag

The backend feature flag is the single source of truth.

- Backend: `APR_POC_ENABLED`
- Default: `false`
- `.env.example` documents `APR_POC_ENABLED=false` and explains it must be intentionally changed to `true` for local APR POC work.
- `NEXT_PUBLIC_APR_POC_ENABLED` was removed from frontend code, `.env.example`, and this documentation.
- `docker-compose.yml` and `docker-compose.dev.yml` pass the backend flag with `APR_POC_ENABLED: ${APR_POC_ENABLED:-false}`.
- The frontend always requests the APR endpoint. A backend `404` displays: `The APR technical proof of concept is disabled in this environment.`

## 10. Python 3.14 local validation limitation

FreeLingo targets Python 3.14. Python 3.14 supports unparenthesized multiple exception types under PEP 758, including repository syntax such as:

```python
except JWTError, KeyError, ValueError:
```

and:

```python
except KeyError, TypeError, ValueError, json.JSONDecodeError:
```

This Codex environment is running Python 3.12. Python 3.12 cannot parse that Python 3.14 PEP 758 exception syntax. Therefore full backend import and pytest validation could not be completed locally in this environment. These lines are not documented here as upstream syntax defects.

`backend/app/core/deps.py` remains on the repository's original Python 3.14 syntax, and `backend/app/services/assessment_voice_trial.py` was not modified.

## 12. GitHub validation

Added `.github/workflows/apr-day2-checks.yml` because the existing broad develop PR workflow is not scoped to APR Day 2 paths.

The focused workflow:

- Runs on pull requests affecting APR backend code, APR frontend code, APR tests, APR documentation, relevant configuration, or the workflow itself.
- Uses Python 3.14.
- Installs backend dependencies from `backend/requirements.txt`.
- Runs `cd backend && pytest tests/test_apr.py -v --no-cov`.
- Uses Node 25, matching the existing develop PR workflow.
- Installs frontend dependencies with `npm ci` from the lockfile.
- Runs frontend lint, TypeScript checking, and focused APR frontend tests.
- Does not add paid services, secrets, PostgreSQL, Redis, Docker, STT, TTS, or LLM calls.

## 13. Tests blocked locally

- Backend dependency installation into `.venv` was blocked by package-index proxy errors: `Tunnel connection failed: 403 Forbidden`.
- Full backend import, backend pytest, and Black formatting safety checks remain blocked in this Codex environment because it uses Python 3.12 and the repository targets Python 3.14 syntax.

## 14. Known limitations

- Backend pytest should be run by the Python 3.14 GitHub workflow before merging.
- The `.venv` exists, but backend dependencies were not fully installed because the package index was blocked by the environment proxy.
- The APR page is intentionally not added to every navigation surface.
- The APR endpoint returns fixed metadata only; it is not a lesson and does not persist learner activity.
- Brazilian Portuguese (`pt-BR`) is represented only as APR metadata in this task; no FreeLingo curriculum package was added.
