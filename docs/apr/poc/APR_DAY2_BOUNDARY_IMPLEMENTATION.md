# APR Day 2 Boundary Implementation

## 1. Goal

Create a small, isolated technical boundary for Academia Português Reconectado (APR) inside FreeLingo without building a lesson, changing FreeLingo learning behavior, or using AI, speech, billing, placement, study-plan, XP, or streak systems.

- `backend/app/apr/schemas.py`

- `frontend/src/app/(app)/apr/primeira-conexao/page.tsx`
- `frontend/tests/app/apr-primeira-conexao.test.tsx`
- `docs/apr/poc/APR_DAY2_BOUNDARY_IMPLEMENTATION.md`

## 3. Existing files modified

- `.env.example`

Note: `backend/tests/test_apr.py` also received a formatter-only line wrap from `./scripts/format.sh`; test behavior did not change.

## 4. Backend route

- Route: `GET /api/apr/modules/primeira-conexao`
- Package: `backend/app/apr/`
- Response: fixed typed metadata for the APR Day 2 technical boundary.
- The response is not generated by AI and does not read lesson, placement, study-plan, progress, billing, conversation, STT, or TTS state.

## 5. Authentication behavior

The route uses FreeLingo's existing authenticated-user dependency, `get_current_user`. This proves APR can reuse account login without reusing CEFR placement or study-plan generation.

Unauthenticated requests receive the existing authentication failure behavior from FreeLingo's auth dependency.

## 6. Frontend route

- Browser route: `/apr/primeira-conexao`
- File: `frontend/src/app/(app)/apr/primeira-conexao/page.tsx`
- The page is inside the existing authenticated application route group, so it reuses the current app authentication wrapper.
- The page calls `GET /api/apr/modules/primeira-conexao` through the existing `apiFetch` helper.
- The page displays loading, success, and technical error states.

## 7. Whether a feature flag was added

A minimal feature flag was added because it fit the existing configuration pattern without a new feature-flag system.

- Backend: `APR_POC_ENABLED`
- Frontend: `NEXT_PUBLIC_APR_POC_ENABLED`
- Both are documented in `.env.example`.
- Default value in code and example configuration is enabled for this proof-of-concept branch.
- When disabled, the backend APR endpoint returns `404`, and the frontend page shows a technical disabled message instead of presenting APR as an ordinary learning product.

## 8. Systems deliberately not used

APR Day 2 deliberately does not use:

- CEFR placement
- Generated study plans
- Lesson generation
- XP
- Streaks
- Subscription or billing checks
- Quotas
- LLMs or prompts
- Speech-to-text
- Text-to-speech
- Pipecat
- WebSockets
- Conversation pipeline
- Progress scoring
- Evidence tables
- Database writes
- Database migrations
- Paid APIs

## 9. Python 3.14 syntax validation limitation

FreeLingo targets Python 3.14. Python 3.14 supports unparenthesized multiple exception types under PEP 758, including repository syntax such as:

```python
except JWTError, KeyError, ValueError:
```

and:

```python
except KeyError, TypeError, ValueError, json.JSONDecodeError:
```

This Codex environment is running Python 3.12. Python 3.12 cannot parse that Python 3.14 PEP 758 exception syntax. Therefore full backend import and pytest validation could not be completed in this environment. These lines are not documented here as upstream syntax defects.

`backend/app/core/deps.py` was restored to the repository's original Python 3.14 syntax, and `backend/app/services/assessment_voice_trial.py` was not modified.

## 11. Tests blocked

- Backend dependency installation into `.venv` was blocked by package-index proxy errors: `Tunnel connection failed: 403 Forbidden`.
- Full backend import, backend pytest, and Black formatting safety checks remain blocked in this Codex environment because it uses Python 3.12 and the repository targets Python 3.14 syntax.

## 12. Known limitations

- Backend pytest should be run in a Python 3.14 environment with backend dependencies installed before merging.
- The `.venv` exists, but backend dependencies were not fully installed because the package index was blocked by the environment proxy.
- The APR page is intentionally not added to every navigation surface.
- The APR endpoint returns fixed metadata only; it is not a lesson and does not persist learner activity.
- Brazilian Portuguese (`pt-BR`) is represented only as APR metadata in this task; no FreeLingo curriculum package was added.
